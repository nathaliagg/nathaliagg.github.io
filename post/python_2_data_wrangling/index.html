Python - 2 - Data Wrangling

Sat, Jan 2, 2021 2-minute read

I use Python daily, and I spend a lot of time doing data wrangling.

What’s data wrangling? Data wrangling is a process of transforming your data from a raw format to a format that is more appropriate for downstream analysis. You’ll have to assess the quality of the current raw state of the data, and make decisions about how you’ll go about improving its quality.

The time you’ll spend in data wrangling will depend on the quality of the data you get. For me, this is often the longest process. When working with data that are given to me, I spend time exploring the data and the metadata (if available), e.g., size of data and data types present. Then, I move on to assess how data are formatted as I often have to deal with spaces where there shouldn’t be any, formatting inconsistencies, missing-values, etc. After all of that, I finally proceed to preprocessing it for my downstream analysis.

In this notebook, I demonstrate how I typically perform data wrangling. For me, I perform data wrangling in a two-step process: exploratory, and data wrangling. In the exploratory step, I retrieve information about the data such as dataframe size, data types in each column, and missing-value count. Sometimes, if appropriate, I’ll obtain descriptive statistics such as total count of values, mean, mininimum, maximum values in each column. Finally, I perform data wrangling depending on which downstream analysis I have to carry out. I demonstrate the typical operations I perform during data wrangling such as renaming columns and rows, reordering columns, converting data types, pivot tables, and split/join columns.

Hope you enjoy!
